# create-pr

현재 feature 브랜치를 dev로 PR 생성한다.

## 입력

`/create-pr <이슈번호>` 형식으로 실행한다 (예: `/create-pr 1`).
이슈 번호가 없으면 사용자에게 묻는다.

## 실행 순서

1. **브랜치 확인**
   - `git branch --show-current` 로 현재 브랜치 확인
   - `dev`, `main` 브랜치이면 경고 후 중단한다

2. **원격 브랜치 push**
   - `git push -u origin <브랜치명>` 실행
   - 이미 push된 경우 그대로 진행

3. **docs-sync 확인**
   - `git log --oneline dev..HEAD` 로 이 브랜치의 커밋 목록을 확인한다
   - `docs:` 커밋이 없고 docs/ 관련 변경도 없으면 아래를 출력하고 중단한다:
     ```
     ⚠ docs 업데이트가 확인되지 않습니다.
     /docs-sync 를 먼저 실행하세요.
     ```
   - 사용자가 "docs 변경 없음"을 명시적으로 확인하면 계속 진행한다

4. **이슈 + 작업 계획 읽기**
   - `gh issue view <N> --comments` 로 이슈 본문과 작업 계획 코멘트를 읽는다

5. **PR 제목 / 본문 작성 및 확인**

   제목: 이슈 제목 그대로 사용

   본문 형식:
   ```
   ## 구현 내용

   (작업 계획의 구현 범위를 바탕으로 2-4줄 요약)

   ## 완료 기준

   (작업 계획의 완료 기준을 체크리스트로 변환)
   - [ ] 항목 1
   - [ ] 항목 2

   Closes #N
   ```

   작성된 제목과 본문을 출력한 뒤 사용자에게 묻는다:
   > "이 내용으로 PR을 생성할까요?"

   사용자가 수정을 요청하면 반영 후 다시 출력한다. 승인하면 6번으로 진행한다.

6. **PR 생성**
   - `gh pr create --base dev --title "<제목>" --body "<본문>"` 실행
   - 생성된 PR URL을 출력한다
