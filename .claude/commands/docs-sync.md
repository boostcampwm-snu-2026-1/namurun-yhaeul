# docs-sync

구현 결과와 docs(설계 문서)의 불일치를 찾아 업데이트한다.  
`/implement` 마지막 단위 완료 후 실행하거나, 구현 중 설계가 바뀌었을 때 언제든 독립 실행 가능.

## 실행 순서

1. **변경 범위 파악**
   - `git diff dev...HEAD` 로 이 브랜치에서 변경된 파일 목록을 확인한다
   - 변경 파일이 없으면 "변경된 파일이 없습니다"를 출력하고 종료한다

2. **관련 docs 파일 파악**
   - `CLAUDE.md`의 `## 상세 문서` 섹션을 확인해 docs 파일 목록을 읽는다
   - 변경된 코드와 관련 있는 docs 파일을 모두 읽는다

3. **불일치 검토**
   아래 항목을 기준으로 코드와 docs를 비교한다:
   - 구현 중 확정된 설계 결정이 docs에 반영되어 있는가?
   - docs에 명시된 구조·타입·API가 실제 구현과 다른가?
   - docs에 없지만 추가된 중요한 컴포넌트·훅·유틸이 있는가?

4. **결과 출력**

   업데이트가 필요한 경우:
   ```
   ## Docs 업데이트 필요

   ### `docs/xxx.md`
   - 변경할 내용 요약 (한 줄씩)

   수정할까요?
   ```

   업데이트가 불필요한 경우:
   ```
   ## Docs 동기화 완료
   코드와 docs 사이에 불일치 없음.
   ```

5. **사용자 승인 시 수정**
   - 해당 docs 파일을 직접 수정한다
   - 수정 후 `/commit-msg` 로 `docs:` 커밋할 것을 안내한다
